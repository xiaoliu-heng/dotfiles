# Dotfiles

![Ubuntu](https://img.shields.io/badge/Ubuntu-supported-E95420?style=flat-square&logo=ubuntu&logoColor=white)
![Arch Linux](https://img.shields.io/badge/Arch-supported-1793D1?style=flat-square&logo=archlinux&logoColor=white)
![No sudo](https://img.shields.io/badge/no%20sudo-user--local%20install-22c55e?style=flat-square)

A small, portable workstation bootstrap for shell, tmux, and git workflows.

It is designed for the common "new machine, make it usable fast" path:
clone the repo, run one script, get the same terminal ergonomics everywhere.

## Highlights

- Installs or reuses `tmux`, `lazygit`, `git-delta`, `zsh`, and `git`
- Installs Oh My Zsh without launching an interactive shell
- Adds zsh-users plugins:
  - `zsh-autosuggestions`
  - `zsh-syntax-highlighting`
  - `zsh-completions`
- Keeps `~/.zshrc` out of the repository
- Edits `~/.zshrc` in place after Oh My Zsh is installed
- Uses symlinks only for portable app configs
- Falls back to user-local binaries when sudo is unavailable
- Keeps private machine settings in `~/.zshrc.local`

## Supported Systems

| System | Preferred install path | User-local fallback |
| --- | --- | --- |
| macOS | Homebrew + `Brewfile` | direct binaries under `~/.local/bin`; optional micromamba |
| Ubuntu | `apt-get` | direct binaries under `~/.local/bin`; optional micromamba |
| Arch Linux | `pacman` | direct binaries under `~/.local/bin`; optional micromamba |

The lightweight fallback downloads release binaries for tools that support it,
such as `lazygit` and `delta`, into `~/.local/bin`. If required tools still
cannot be installed that way, the installer asks before using micromamba under
`~/.local/share/dotfiles`.

## Quick Start

```sh
git clone <your-repo-url> ~/workspace/dotfiles
cd ~/workspace/dotfiles
./install.sh
```

The installer is idempotent. Existing tmux and lazygit config files are moved to
`~/.dotfiles-backup/<timestamp>/` before symlinks are created.

## What Gets Managed

| Target | Source | Strategy |
| --- | --- | --- |
| `~/.tmux.conf` | `dotfiles/tmux.conf` | symlink |
| `~/.config/lazygit/config.yml` | `config/lazygit/config.yml` | symlink |
| `~/.zshrc` | generated by Oh My Zsh | edited in place |
| `~/.zshrc.local` | local only | created if missing, never tracked |

## tmux Setup

The tmux config includes:

- `C-a` prefix
- mouse support
- vi-style copy mode
- current-directory splits and new windows
- pane navigation with `h/j/k/l`
- resize bindings with `H/J/K/L`
- popup shell with `prefix + t`
- lazygit popup with `prefix + g`
- a plugin-free statusline theme

## Private Settings

Do not commit tokens, private URLs, or machine-specific settings. Put them in:

```sh
~/.zshrc.local
```

Example:

```sh
export EXAMPLE_SERVICE_URL="https://example.com"
export EXAMPLE_SERVICE_TOKEN="..."
```

The repository ignores local override files by default.

## Safety Notes

- `~/.zshrc` is not stored in this repository.
- The installer only owns a marked block inside `~/.zshrc`.
- Existing managed config files are backed up before symlinking.
- User-local binaries live under `~/.local/bin`.
- micromamba installs, when explicitly accepted, live under `~/.local/share/dotfiles`.
- Secrets belong in `~/.zshrc.local`, not in git.
